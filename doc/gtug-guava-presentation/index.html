Wprowadzenie

Collections a.k.a. Google Collections Library

  • Guava wproawdza kilka nowych kolekcji, których nie ma w JDK
  • Zaprojektowane by koegzystować z typami JDK
    • Nowe kolekcje (Multimap, Multiset, BiMap ...)
    • Immutable collections
    • Utilities (Iterables, Iterators ...)

Multiset

    Do czego służy?

  • Możliwe duplikaty, kolejność istotna = List
  • Bez duplikatów, kolejność nieistotna = Set
  • Duplikaty, kolejność nieistotna = Multiset (Guava)

    Cechy

  • Idealny do histogramów
  • Dodawanie wielu elementów (na raz)
  • Usuwanie wielu elementów (na raz)
  • Bez ujemnych liczników

Multiset - samodzielnie

Problemy

  • Sprawadzanie wartości nie nullowych
  • Usuwanie wartości nieobecnych
  • Obliczanie rozmiaru zbioru

Multiset - Guava

Multimap

  • Tak jak mapa ale klucze mogą się powtarzać { a => 1, a => 2, b => 3 }
  • Można rozważać jako kolekcję kolekcji { a => [1,2], b => [3] }

Multimap - samodzielnie

    Problemy przy implementacji

  • Sprawdzanie wartości nie nullowych
  • Inicjalizacja pustej kolekcji

Multimap - Guava

Immutables

    Po co?

  • Bezpiecznie
  • Wielowątkowo
  • Szybko
  • Zoptymalizowanie pamięciowo

    Pamiętajmy o:

  • Kolekcja jest niezmienna a nie jej elementy!

    Zyski

  • Guava - zoptymalizowany do bycia niezmiennym
  • JDK wrappery są nie do końca niezmienne
  • Kod z JDK jest zbyt dosłowny
